### Guion de Video - Trabajo Final Integrador: App de Reservas

**Duración total:** 8 minutos
**Integrantes:** Juan Sommaruga, Frank Fobello, Alan Garcia Talma, Alexis Nehuen Checura.

---

#### **Parte 1: Introducción y Arquitectura (2 minutos) - Presenta Juan Sommaruga**

**(0:00 - 0:30) Introducción**

*   **Juan:** "Hola a todos, somos estudiantes de Programación III y hoy les presentamos nuestro trabajo final integrador: una API REST para la gestión de reservas de salones de eventos. El objetivo de este proyecto es ofrecer una solución robusta y escalable para administrar salones, servicios adicionales, usuarios y reservas de manera eficiente."

**(0:30 - 1:30) Arquitectura y Tecnologías**

*   **Juan:** "Para construir esta API, elegimos un stack de tecnologías moderno y confiable basado en JavaScript. Como entorno de ejecución utilizamos Node.js, que nos permite tener un servidor rápido y eficiente. El corazón de nuestra aplicación es Express, un framework minimalista que nos facilitó la creación de las rutas y la gestión de las peticiones HTTP."
*   **Juan:** "En cuanto a la base de datos, optamos por MySQL, un sistema de gestión de bases de datos relacional muy popular y potente. La conexión con la base de datos desde Node.js la realizamos con la librería `mysql2`."
*   **Juan:** "La arquitectura de nuestro proyecto está organizada en capas, separando las responsabilidades. Tenemos una capa de rutas que define los endpoints de la API, una capa de controladores que maneja la lógica de negocio, una capa de servicios para la lógica más específica y una capa de acceso a datos que se comunica con la base de datos."

**(1:30 - 2:00) Documentación y Pruebas**

*   **Juan:** "Para asegurar que nuestra API sea fácil de entender y utilizar, implementamos documentación interactiva con Swagger. Esto permite a cualquier desarrollador explorar los endpoints, ver los parámetros que aceptan y probarlos directamente desde el navegador. Para la validación de los datos de entrada, utilizamos `express-validator`, garantizando que solo procesamos información correcta y segura."

---

#### **Parte 2: Gestión de Salones y Servicios (2 minutos) - Presenta Frank Fobello**

**(2:00 - 3:00) Gestión de Salones**

*   **Frank:** "Ahora, profundicemos en las funcionalidades principales. Una de las entidades centrales de nuestro sistema son los 'Salones'. Nuestra API permite realizar un CRUD completo (Crear, Leer, Actualizar y Borrar) sobre los salones."
*   **Frank:** "Podemos crear un nuevo salón especificando su nombre, dirección, capacidad y el importe del alquiler. También podemos obtener un listado de todos los salones, con opciones de filtrado por capacidad o precio, paginación para manejar grandes cantidades de datos, y ordenamiento por diferentes campos. Por supuesto, también es posible obtener los detalles de un salón específico por su ID, actualizar su información y realizar un borrado lógico."

**(3:00 - 4:00) Gestión de Servicios Adicionales**

*   **Frank:** "Además de los salones, la aplicación permite gestionar 'Servicios' adicionales que los clientes pueden contratar para sus eventos. Al igual que con los salones, hemos implementado un CRUD completo para los servicios."
*   **Frank:** "Se pueden crear servicios como 'catering', 'decoración' o 'DJ', cada uno con su descripción y costo. Los clientes pueden agregar estos servicios a sus reservas, y el sistema calculará el importe total automáticamente. Esta funcionalidad añade un gran valor a la aplicación, permitiendo una personalización completa de los eventos."

---

#### **Parte 3: Gestión de Usuarios y Reservas (2 minutos) - Presenta Alan Garcia Talma**

**(4:00 - 5:00) Gestión de Usuarios**

*   **Alan:** "Ahora hablemos de los 'Usuarios'. El sistema soporta diferentes tipos de usuarios: 'cliente', 'empleado' y 'administrador', cada uno con distintos niveles de permisos. La API permite registrar nuevos usuarios, obtener sus datos, actualizarlos y realizar un borrado lógico."
*   **Alan:** "La gestión de usuarios es fundamental para la seguridad y la personalización de la experiencia. Los clientes pueden ver y gestionar sus propias reservas, mientras que los administradores tienen acceso a toda la información del sistema."

**(5:00 - 6:00) Creación y Gestión de Reservas**

*   **Alan:** "El núcleo de la aplicación es, por supuesto, la gestión de 'Reservas'. Un cliente puede crear una reserva asociando un salón, una fecha, un turno y los servicios adicionales que desee. El sistema verifica la disponibilidad del salón en la fecha y turno seleccionados para evitar conflictos."
*   **Alan:** "Al crear la reserva, se calcula el importe total, sumando el costo del salón y los servicios seleccionados. Además, implementamos un servicio de notificación por correo electrónico utilizando `Nodemailer` y plantillas de `Handlebars` para enviar una confirmación de la reserva al usuario, mejorando la experiencia del cliente."

---

El proyecto cumple con todos los requisitos planteados, aplicando los conceptos aprendidos en la materia como arquitectura en capas, validaciones, y el uso de herramientas profesionales como Node.js, Express y MySQL."
*   **Alexis:** "Este proyecto fue un gran desafío que nos permitió consolidar nuestros conocimientos y trabajar en equipo de manera efectiva, utilizando GitHub para el control de versiones y la colaboración."
